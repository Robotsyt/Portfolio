AD Virtual Machine

  • Tech Stack: Windows Server (Active Directory, GPO, File Services), Windows 10/11, and Hyper-V / VMware Virtualization.
  • Github URL: Project Link

In my Windows Server Home Lab Project, I built a fully functional, on-premises enterprise network environment from scratch using virtualization. I began by deploying and configuring a root Windows Server Domain Controller (DC=CONTOSO, DC=COM) to establish a centralized identity management hub. From there, I designed a structured Organizational Unit (OU) framework to systematically organize and manage our users, computer endpoints, and dedicated IT security groups. Once the identity database was online, I implemented Group Policy Objects (GPOs) to enforce automated, domain-wide security profiles and strict password policies. To demonstrate complete end-to-end functionality, I joined a Windows workstation to the domain and successfully verified that my corporate governance rules applied directly to the client machine. Finally, I set up corporate File Services to build centralized network shares, configuring granular NTFS and Share permissions to ensure data access is strictly restricted by department.
